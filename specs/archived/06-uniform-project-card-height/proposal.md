# 统一项目卡片高度

## 问题描述

项目列表页中，项目卡片使用 CSS Grid 3 列布局。由于描述字段为条件渲染：

- 有描述的项目卡片：标题 + 描述文字（`line-clamp-2`）+ 统计信息
- 无描述的项目卡片：标题 + 统计信息

导致有描述的卡片比无描述的卡片更高，在网格中视觉上参差不齐。

## 边界场景分析

| # | 场景 | 当前表现 | 影响 |
|---|------|----------|------|
| 1 | **描述为空（无 description 字段）** | 卡片缺少描述行，高度最小 | 基础问题 |
| 2 | **描述为空字符串 `description: ""`** | `""` 为 falsy，条件渲染跳过，同上 | 同 1 |
| 3 | **描述很短（1 行）** | 只占 1 行高度，低于 2 行描述的卡片 | 网格行内仍不对齐 |
| 4 | **描述很长（超过 2 行）** | `line-clamp-2` 截断，固定 2 行视觉高度 | 用户无法看到完整的描述内容 |
| 5 | **描述刚好 2 行** | 完整显示，与 `line-clamp-2` 占位一致 | 正常显示 |
| 6 | **项目名称很长** | `truncate` 单行截断，高度固定 | 不影响高度 |
| 7 | **项目名称很短（1-2 字）** | 单行显示 | 不影响高度 |
| 8 | **统计信息为 0（table_count=0）** | 仍渲染图标+文字，占位固定 | 不影响高度 |
| 9 | **所有项目都无描述** | 所有卡片一致，无视觉问题 | 性能无退化 |
| 10 | **所有项目都有描述且行数一致** | 所有卡片一致 | 不需要处理 |
| 11 | **响应式断点切换（1/2/3 列）** | Grid 列数变化，每行最高卡片动态变化 | 需要适配 |
| 12 | **浏览器缩放 200%** | rem 缩放，行高放大 | `min-h` 需用 rem 单位自适应 |
| 13 | **操作系统字体不同** | Tailwind 固定 line-height 保证一致 | 不影响 |
| 14 | **列表含大量项目（>100）** | 卡片 DOM 多，不应引起布局抖动（reflow） | 避免频繁重排 |
| 15 | **无障碍屏幕阅读器** | 空占位 div 不应被朗读 | 占位元素不应含可访问文本 |
| 16 | **描述超长，用户需要阅读完整内容** | 被截断后无法获取完整信息 | 需要悬浮提示查看全文 |
| 17 | **描述较短（不触发 line-clamp-2）但有 tooltip** | tooltip 显示与可见内容重复 | 可接受，不影响功能 |

## 解决方案

### 核心策略：两层防御

1. **flex 列 + h-full** — 卡片容器使用 `flex flex-col h-full`，填满 grid cell 高度
2. **描述区占位保留** — 无论是否有描述，描述区域的垂直空间保持一致
3. **mt-auto 固定底部** — 统计信息栏始终固定在卡片底部

### 具体改动

```tsx
// ProjectCard.tsx
<Link
  to={`/projects/${project.id}`}
  className="block rounded-lg border bg-white p-5 shadow-sm transition hover:shadow-md hover:border-blue-200 no-underline flex flex-col h-full"
>
  <h3 className="text-base font-semibold text-gray-900 mb-2 truncate"
      title={project.name}>
    {project.name}
  </h3>
  {/* 描述区域：始终占用固定最小高度，保证卡片等高 */}
  <div className="min-h-[2.5rem] mb-3">
    {project.description && (
      <p className="text-sm text-gray-500 line-clamp-2"
         title={project.description}>
        {project.description}
      </p>
    )}
  </div>
  {/* 统计信息：mt-auto 确保在卡片底部对齐 */}
  <div className="flex items-center gap-4 text-xs text-gray-400 mt-auto">
    ...
  </div>
</Link>
```

### 关键设计决策

- **`min-h-[2.5rem]`**: `text-sm` 在 Tailwind 中 line-height 为 `1.25rem`（20px），2 行 = `2.5rem`（40px）。使用 rem 单位，随浏览器缩放自适应。
- **描述为空时占位元素无文本内容**：屏幕阅读器不会读到多余信息。
- **既有卡片不迁移**：仅修改 ProjectCard 组件，无数据迁移成本。
- **`title` 属性悬浮提示**：使用原生 HTML `title` 属性，最轻量的方式在悬浮时显示完整文本。零依赖、零运行时开销，浏览器提供原生 tooltip。无需条件判断是否溢出——短文本时重复显示无害。

### 不建议的方案与理由

| 方案 | 问题 |
|------|------|
| `h-[180px]` 固定卡片高度 | 响应式下不同列宽需要不同的高度硬编码 |
| JS 动态计算最高卡片 | 引入运行时开销，且初始渲染有布局抖动 |
| 删除描述显示 | 破坏功能，用户需要看到描述 |
| 仅用 `flex-1` 弹性填充 | 无描述卡片内，名字下方大片空白与有描述卡片视觉不同 |

## 业务规则

1. 所有项目卡片在网格中必须等高
2. 有描述的项目正常显示描述文本（最多 2 行）
3. 底部统计信息栏在所有卡片中垂直位置一致
4. 描述为空时，保留的空间不可见（无多余 UI 元素）
5. 描述文本被 `line-clamp-2` 截断时，鼠标悬浮可查看完整内容

## 约束条件

- 仅改动前端代码（`ProjectCard.tsx`），不涉及后端
- 使用 Tailwind CSS 实现，不引入额外 CSS
- 保持响应式布局（`sm:grid-cols-2 lg:grid-cols-3`）

## 成功标准

- [ ] 有描述和无描述的项目卡片在网格行内完全等高
- [ ] 底部统计信息栏在所有卡片中垂直位置一致
- [ ] 描述文本溢出时仍保持 `line-clamp-2` 截断
- [ ] 描述超长时，鼠标悬浮显示原生 tooltip 完整内容
- [ ] 描述较短时，鼠标悬浮不干扰显示
- [ ] 描述为空的卡片，保留空间不影响交互（点击穿透正常）
- [ ] 浏览器缩放 200% 时卡片仍等高
- [ ] 不改变现有交互行为（点击跳转、hover 效果等）
- [ ] 现有测试通过
