# 项目列表分页调整为每页 12 条

## 问题描述

当前项目列表每页显示 20 个项目，卡片在 3 列网格布局下视觉效果偏密，用户浏览负担较重。调整为每页 12 条（3 列 × 4 行），信息密度更适中。

## 业务规则

1. 每页固定显示 12 个项目
2. 分页控件（上一页/下一页、页码信息）自动适配新的 page size
3. 不改变其他列表类接口的分页行为

## 约束条件

- 前端硬编码的 page size 和后端默认值均需同步修改
- 不影响其他 API 调用方（目前仅前端使用）

## 涉及改动

| 位置 | 文件 | 当前值 | 目标值 |
|------|------|--------|--------|
| 前端 | `frontend/src/pages/ProjectList.tsx` | `size = 20` | `size = 12` |
| 后端 | `backend/app/api/projects.py` | `size: int = 20` | `size: int = 12` |

## 边界场景

| # | 场景 | 预期 |
|---|------|------|
| 1 | 项目总数 < 12 | 不分页，正常显示 |
| 2 | 项目总数 = 12 | 恰好一页，不显示分页控件 |
| 3 | 项目总数 = 13 | 第 1 页 12 条，第 2 页 1 条 |
| 4 | 项目总数 = 24 | 恰好 2 页 |
| 5 | 搜索过滤后 < 12 | 客户端过滤不影响分页逻辑 |
| 6 | 切换到最后一页后删除项目 | 当前页码超出总页数时，回退到上一页 |
| 7 | 新建项目后 | 列表刷新，总数 +1，分页自动重算 |

## 成功标准

- [ ] 项目列表每页显示 12 个项目，恰好 3 列 × 4 行
- [ ] 分页控件页码信息正确（"第 X / Y 页，共 Z 个"）
- [ ] 搜索过滤不受影响
- [ ] 新建/删除项目后列表刷新正常
- [ ] 后端 `size` 参数仍可通过 query string 覆盖（向后兼容）
