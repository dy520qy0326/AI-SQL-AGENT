# PRD: 可视化模块 (Visualization)

## 1. 产品概述

可视化模块负责将解析出的表结构和关系以图形化方式呈现给用户，核心是交互式 ER 图（实体关系图），辅助用户快速理解数据库架构。

## 2. 目标用户

- **主要用户**：开发人员、DBA、数据分析师
- **次要用户**：技术管理者、数据治理人员

## 3. 功能需求

### P0 — ER 关系图

| 功能 | 说明 |
|------|------|
| 节点展示 | 每张表为一个节点，节点内显示字段列表（带图标区分 PK/FK） |
| 关系连线 | 表之间的关联关系以连线表示，箭头指向目标表 |
| 连线标注 | 连线上标注关联列名，如 `user_id → id` |
| 关系类型区分 | 实线 = 显式外键，虚线 = 推断关系，颜色区分置信度 |
| 拖拽布局 | 节点可自由拖拽，自动力布局作为初始排列 |
| 缩放/平移 | 支持大图浏览 |

### P1 — 交互操作

| 功能 | 说明 |
|------|------|
| 节点展开/折叠 | 默认显示字段名，点击展开显示完整类型/non-null/默认值 |
| 点击高亮 | 点击表节点，高亮其所有关联路径（一度/二度关系） |
| 搜索定位 | 输入表名快速定位并居中到该节点 |
| 迷你地图 | 大图时右下角显示缩略导航 |
| 关系筛选 | 按关系类型（外键/推断/AI）/置信度筛选显示连线 |
| 导出图片 | 导出当前视图为 PNG/SVG |

### P2 — 辅助视图

| 功能 | 说明 |
|------|------|
| 列表视图并排 | 右侧面板展示当前选中表的完整 schema |
| 全屏模式 | 隐藏侧边栏，最大化图形区域 |
| 对比视图 | 两个项目/两个版本的 ER 图左右对比 |
| 布局预设 | 按星型模型/雪花模型自动排布 |

## 4. 渲染性能目标

| 规模 | 目标 |
|------|------|
| < 50 张表 | 即时加载 (< 500ms) |
| 50~200 张表 | 流畅交互 (> 30fps) |
| 200~500 张表 | 首次渲染 < 3s, 可交互 |
| > 500 张表 | 懒加载 + 分组折叠 |

## 5. 技术选型建议

| 方案 | 适用场景 | 理由 |
|------|---------|------|
| vis-network | 主选 | 性能好，支持力布局、事件丰富、中文文档多 |
| D3.js force-graph | 定制化需求 | 灵活性最高，但开发量大 |
| Mermaid ER 图 | 内嵌导出 | 适合静态展示，不支持交互 |
| Cytoscape.js | 200+ 表 | 专业图处理，性能优于 vis-network |

**推荐**：前端交互图用 **vis-network**，导出用 **Mermaid**。

## 7. 验收标准

- [ ] 上传 10 张表的 DDL，关系图展示完整正确
- [ ] 节点可拖拽，布局不重置
- [ ] 点击节点高亮关联路径（一度关系）
- [ ] 连线区分实线（外键）和虚线（推断）
- [ ] 搜索框输入表名即时定位
- [ ] 导出 PNG 清晰可读
- [ ] 50 张表交互帧率 > 30fps
